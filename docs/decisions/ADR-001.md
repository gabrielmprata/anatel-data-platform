# ADR-001 — Arquitetura Inicial da Plataforma

**Status:** Aprovado

**Data:** 08/07/2026

---

# Contexto

O objetivo deste projeto é construir uma plataforma de Engenharia de Dados capaz de automatizar a obtenção, tratamento e disponibilização dos dados públicos da ANATEL.

Inicialmente a plataforma será utilizada para os dados de Banda Larga Fixa, porém sua arquitetura deverá permitir expansão futura para outros painéis disponibilizados pela ANATEL.

A plataforma deverá atender aos seguintes objetivos:

* Automatizar o download dos dados públicos.
* Eliminar processos manuais de importação de arquivos.
* Possibilitar integração com bancos de dados.
* Disponibilizar dados para consumo no Power BI.
* Facilitar manutenção e evolução do projeto.

---

# Problema

Em projetos de análise de dados é comum concentrar toda a lógica em notebooks.

Embora seja suficiente para estudos exploratórios, essa abordagem apresenta algumas limitações:

* código pouco reutilizável;
* difícil manutenção;
* baixa escalabilidade;
* baixa testabilidade;
* dificuldade para automatização.

---

# Decisão

Adotar uma arquitetura modular baseada em responsabilidades únicas.

Cada módulo será responsável por apenas uma atividade dentro do processo ETL.

Exemplo:

* Configuração
* Download
* Extração
* Transformação
* Validação
* Exportação
* Representação dos dados

Os notebooks serão utilizados apenas para exploração e análise dos dados.

Toda a lógica da aplicação ficará concentrada na biblioteca Python.

---

# Consequências

## Benefícios

* Código reutilizável.
* Fácil manutenção.
* Facilidade para criação de testes automatizados.
* Integração com GitHub Actions.
* Integração com PostgreSQL.
* Facilidade para expansão da plataforma.

## Desvantagens

* Maior esforço inicial de organização.
* Mais arquivos do que uma solução baseada apenas em notebooks.
* Curva de aprendizado maior para quem está iniciando.

---

# Decisão Final

A plataforma será desenvolvida como uma biblioteca Python modular, desacoplando a lógica de ETL dos notebooks de exploração de dados.

Essa arquitetura servirá como base para futuras integrações, automações e expansão do projeto.
